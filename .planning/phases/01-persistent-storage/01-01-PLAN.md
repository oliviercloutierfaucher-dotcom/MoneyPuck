Output: Updated database.py with Railway volume awareness, updated Dockerfile, TTL cache in web_preview.py, and comprehensive tests for all changes.

  <action>
1. In `app/web/web_preview.py`, add a `TTLCache` class near the top (after imports):
   ```python
   class TTLCache:
       """Simple in-memory cache with per-key TTL."""
       def __init__(self, ttl_seconds: float = 90.0):
           self._ttl = ttl_seconds
           self._store: dict[str, tuple[float, Any]] = {}
       def get(self, key: str) -> Any | None:
           entry = self._store.get(key)
           if entry is None:
               return None
           ts, value = entry
           if time.monotonic() - ts > self._ttl:
               del self._store[key]
               return None
           return value
       def set(self, key: str, value: Any) -> None:
           self._store[key] = (time.monotonic(), value)
   ```
   - Import `time` at the top (add to existing imports)
   - Import `Any` from typing (likely already imported or available)

2. Create a module-level cache instance: `_snapshot_cache = TTLCache(ttl_seconds=90.0)`

3. In the request handler's dashboard/API paths that call `build_market_snapshot()`:

   - Check `_snapshot_cache.get(cache_key)` first

   - If cache miss or force refresh: call `build_market_snapshot()`, then `_snapshot_cache.set(cache_key, snapshot)`
   - Log cache hits at debug level: `log.debug("Cache hit for %s", cache_key)`

4. Support `?refresh=1` query parameter to bypass cache (per RESEARCH.md recommendation)
