<task type="checkpoint:human-action" gate="blocking">
  <name>Task 1: Configure Railway volume and env var</name>
  <files></files>
  <action>
The user must configure two things in the Railway dashboard:

1. **Add volume mount:**
   - Railway Dashboard -> Select MoneyPuck service -> Settings -> Volumes
   - Click "Mount Volume" (or "Add Volume")
   - Set mount path to: `/data`
   - Save

2. **Set RAILWAY_RUN_UID=0:**
   - Railway Dashboard -> Select MoneyPuck service -> Variables
   - Add new variable: `RAILWAY_RUN_UID` = `0`
   - This tells Railway to run the container as root, matching volume ownership
   - Without this, SQLite gets SQLITE_READONLY errors because appuser cannot write to root-owned volume

3. **Deploy** the latest code (which includes Plan 01 changes)
  </action>
  <verify>User confirms volume and env var are configured</verify>
  <done>Railway volume mounted at /data, RAILWAY_RUN_UID=0 set, deploy completed</done>
</task>
