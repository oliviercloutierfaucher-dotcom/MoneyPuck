1. **Refactor app.py page routes** to add dual-mode handlers for all 5 tabs:

   Replace the existing `dashboard_page` route with:

   ```python
   @app.get("/", response_class=HTMLResponse)
   @app.get("/games", response_class=HTMLResponse)
   @app.get("/index.html", response_class=HTMLResponse)
   def games_page(request: Request, demo: str = "0", region: str = "qc",
                  bankroll: float = 1000.0, min_edge: float = 2.0, min_ev: float = 0.02):
       params = _make_params(region=region, bankroll=bankroll, min_edge=min_edge, min_ev=min_ev)
       data = _build_demo_dashboard(params) if _is_demo(demo) else _build_live_dashboard(params)
       data_json = json.dumps(data).replace("</", r"<\/")

       if request.headers.get("HX-Request") == "true":
           return templates.TemplateResponse(
               request=request,
               name="partials/games.html",
               context={"data": data, "data_json": data_json},
           )
       return templates.TemplateResponse(
           request=request,
           name="base.html",
           context={"data": data, "data_json": data_json, "active_tab": "games"},
       )
   ```

   Add similar handlers for:
   - `GET /value-bets` -- uses same `_build_demo_dashboard` / `_build_live_dashboard` data
   - `GET /arbs` -- uses same dashboard data (arbs are in the dashboard payload)
   - `GET /performance` -- uses `_build_performance_data()` for its own data, plus `_build_demo_dashboard()` or `_build_live_dashboard()` if needed for shared context
   - `GET /props` -- uses dashboard data (props are in the payload)

   Each handler follows the same pattern:
   - Accept same query params (demo, region, bankroll, min_edge, min_ev)
   - Check `request.headers.get("HX-Request") == "true"`
   - If HTMX: return `TemplateResponse` with `partials/{tab}.html`
   - If direct: return `TemplateResponse` with `base.html` and `active_tab` set
   - Pass `data_json` for partials that still use client-side JS rendering

   **IMPORTANT:** Do NOT break existing API routes. The `api` router with `/api/dashboard`, `/api/performance`, `/api/opportunities`, `/api/odds-history` must remain exactly as-is.
